# 7.07 — CentOS 4.6 vault RPM

- **Package:** `ghostscript-7.07-33.2.el4_6.1.x86_64.rpm`
- **Source:** CentOS 4.6 vault updates
  (`https://ftp.iij.ad.jp/pub/linux/centos-vault/4.6/updates/x86_64/RPMS/`)
- **Base image:** `centos:6` (glibc 2.12), `.repo` pointed at `vault.centos.org`
- **Method:** RPM payload extracted with `rpm2cpio | cpio`

## Dependency closure

`readelf -d` on the package's `libgs.so.7` shows it needs, among the usual libc
sonames: `libVFlib2.so.24`, `libttf.so.2`, `libpng12.so.0`, `libjpeg.so.62`,
`libz.so.1`, `libXt/SM/ICE/Xext/X11.so.6`, `libcupsimage.so.2`, `libcups.so.2`,
`libgmodule-2.0.so.0`, `libglib-2.0.so.0`, `libstdc++.so.6`.

From CentOS 6 (`yum`): `libpng libjpeg-turbo cups-libs glib2 libstdc++ zlib
libX11 libXext libXt libSM libICE` plus `urw-fonts ghostscript-fonts` and the
`binutils gdb` tools.

Two sonames do not exist on CentOS 6 and are pulled from the CentOS 4 vault:

- `libVFlib2.so.24` — from `VFlib2-2.25.6-25.x86_64.rpm`.
- `libttf.so.2` — FreeType **1.x**. On EL4 the `freetype-2.1.9-6.el4` package
  bundled `libttf.so.2` next to `libfreetype.so.6`. Only `libttf.so.2*` is copied
  out, so CentOS 6's own `libfreetype.so.6` is not clobbered.

## Gotchas

- Without `urw-fonts` + `ghostscript-fonts`, gs 7.07 aborts at startup with
  `Can't find (or can't open) font file n021003l.pfb` ... `Unable to substitute
  for font` (exit 255) *before* running any `-c` program, so the probe appeared
  to "hang"/print nothing until the fonts were installed.
- CentOS 6's `cpio` is old and lacks `-D`; extraction is done with `cd / &&
  rpm2cpio … | cpio -idm`.

## Verification

- `gs --version` → `7.07`
- probe → `integertype` / `5000000000` ⇒ **64-bit** PostScript integers.
