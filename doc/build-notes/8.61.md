# 8.61 — Ubuntu 7.10 "gutsy" deb

- **Packages:** `ghostscript_8.61.dfsg.1~svn8187-0ubuntu3_amd64.deb`,
  `libgs8_8.61.dfsg.1~svn8187-0ubuntu3_amd64.deb`,
  `gsfonts_8.11+urwcyr1.0.7~pre41-3_all.deb`,
  `libgnutls13_1.6.3-1build1_amd64.deb`
- **Source:** Ubuntu old-releases
  (`http://old-releases.ubuntu.com/ubuntu/pool/main/`)
- **Base image:** `debian/eol:lenny` (glibc 2.7)
- **Method:** deb payloads extracted with `dpkg-deb -x`

## Why this one is the largest deb closure

Gutsy split Ghostscript into a `ghostscript` front-end binary plus a `libgs8`
shared library. `/usr/bin/gs` needs `libgs.so.8`, and `readelf -d libgs.so.8`
pulls a long list: `libcupsimage.so.2`, `libcups.so.2`, `libtiff.so.4`,
`libjpeg.so.62`, `libpng12.so.0`, `libgssapi_krb5.so.2`, `libkrb5.so.3`,
`libk5crypto.so.3`, `libcom_err.so.2`, `libgnutls.so.13`, `libpaper.so.1`,
`libfontconfig.so.1`, `libstdc++.so.6`, plus the usual libc set.

## Why lenny + one extracted library

- The binary needs `libc6 >= 2.6`, so etch (glibc 2.3.6) is too old → lenny.
- Lenny supplies almost everything from `apt`: `libcupsimage2 libcups2 libtiff4
  libjpeg62 libpng12-0 libpaper1 zlib1g libstdc++6 libfontconfig1 libkrb53
  libcomerr2 libtasn1-3 libgpg-error0 libgcrypt11`. (`libkrb53` provides the
  three krb5 sonames; `libtasn1-3`/`libgpg-error0`/`libgcrypt11` satisfy the
  gnutls dependency chain.)
- The one soname lenny lacks is `libgnutls.so.13` (lenny shipped gnutls26 →
  `libgnutls.so.26`). It is extracted from the gutsy `libgnutls13` deb.

## Verification

- `gs --version` → `8.61`
- probe → `integertype` / `5000000000` ⇒ **64-bit** PostScript integers.
- `ldd /usr/bin/gs` reports no missing libraries.
