# 8.54 — Debian 4.0 "etch" deb

- **Package:** `gs-gpl_8.54.dfsg.1-5etch2_amd64.deb` (and `gs-common`)
- **Source:** Debian etch archive (`http://archive.debian.org/debian etch main`)
- **Base image:** `debian/eol:etch` (glibc 2.3.6)
- **Method:** **repo-backed `apt install`** — the cleanest of the eight

## Why this is the simplest

8.54 *is* an etch package, and `debian/eol:etch` already points its
`sources.list` at `archive.debian.org`, so it installs natively:

```
apt-get -o Acquire::Check-Valid-Until=false update
apt-get install -y --force-yes binutils gdb gs-gpl gs-common
```

`gs-common` provides the `/usr/bin/gs` alternatives symlink, so no manual
symlink is needed. No package extraction, no soname hunting.

## Gotchas

- The etch `Release` files are long expired: `apt-get update` needs
  `-o Acquire::Check-Valid-Until=false` and `apt-get install` needs
  `--force-yes`, or it refuses the `KEYEXPIRED` repository.
- etch's apt is too old for `--no-install-recommends`; that flag is omitted.

## Choice of source

The TODO also listed an Ubuntu 7.04 feisty alternate
(`gs-gpl_8.54.dfsg.1-5build1_amd64.deb`). The Debian etch build was chosen
because it is a clean repo-backed install; the feisty alternate was skipped.
