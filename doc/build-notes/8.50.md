# 8.50 — Ubuntu 6.10 "edgy" deb

- **Package:** `gs-gpl_8.50-1.1ubuntu1_amd64.deb`
- **Source:** Ubuntu old-releases
  (`http://old-releases.ubuntu.com/ubuntu/pool/main/g/gs-gpl/`)
- **Base image:** `debian/eol:lenny` (glibc 2.7) — **not** etch
- **Method:** deb payload extracted with `dpkg-deb -x`

## Why lenny, not etch

The edgy binary declares `libc6 (>= 2.4-1)`. On etch (glibc 2.3.6) it fails at
load with:

```
gs: /lib/libc.so.6: version `GLIBC_2.4' not found (required by gs)
```

Lenny's glibc 2.7 satisfies `GLIBC_2.4` and still ships every other soname the
binary needs.

## Dependency closure

Real ELF is `/usr/bin/gs-gpl`. `readelf -d` needs: `libpaper.so.1`,
`libXt/SM/ICE/Xext/X11.so.6`, `libm`, `libc`. (No png/jpeg/gimpprint.)

From lenny (`apt`): `libpaper1 libxt6 libsm6 libice6 libxext6 libx11-6`, plus
`binutils gdb wget`.

After extraction: `ldconfig` and `ln -sf /usr/bin/gs-gpl /usr/bin/gs`.

## Verification

- `gs --version` → `8.50`
- probe → `integertype` / `5000000000` ⇒ **64-bit** PostScript integers.
- `ldd /usr/bin/gs-gpl` reports no missing libraries.
