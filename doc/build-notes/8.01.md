# 8.01 — Ubuntu 4.10 "warty" deb

- **Package:** `gs-gpl_8.01-4_amd64.deb`
- **Source:** Ubuntu old-releases
  (`http://old-releases.ubuntu.com/ubuntu/pool/main/g/gs-gpl/`)
- **Base image:** `debian/eol:etch` (glibc 2.3.6)
- **Method:** deb payload extracted with `dpkg-deb -x`

## Dependency closure

The real ELF is `/usr/bin/gs-gpl` (the deb's `/usr/bin/gs` comes from the
separate `gs-common` package). `readelf -d /usr/bin/gs-gpl` needs:
`libpaper.so.1`, `libXt/SM/ICE/Xext/X11.so.6`, `libpng12.so.0`,
`libjpeg.so.62`, `libz.so.1`, `libgimpprint.so.1`, `libm`, `libc`.

From etch (`apt`): `libpaper1 libxt6 libsm6 libice6 libxext6 libx11-6
libpng12-0 libjpeg62 zlib1g` plus `binutils gdb wget`.

`libgimpprint.so.1` is **not** in etch (gimp-print became gutenprint), so the
warty `libgimpprint1_4.2.6-5ubuntu2_amd64.deb` is extracted from the same Ubuntu
pool.

After extraction: `ldconfig` and `ln -sf /usr/bin/gs-gpl /usr/bin/gs`.

## Verification

- `gs --version` → `8.01`
- probe → `integertype` / `5000000000` ⇒ **64-bit** PostScript integers.
- `ldd /usr/bin/gs-gpl` reports no missing libraries.
